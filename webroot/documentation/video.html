video([location])

The video-command embeds a video and plays it on the subject's display.
IMPORTANT: THIS COMMAND IS NOT SUPPORTED AT THE MOMENT AND MAY THEREFORE NOT WORK AS YOU EXPECT.
Tip: You might want to combine this command with the waitTime-command in order to advance the experiment after the video has finished playing.
In order to use this command you need to check the "Use JMF"-option at the start of the experiment.
Unfortunately not all video and audio are supported by the Java Media Framework, which is used to render the videos. However, you can convert your videos to a supported format.
For a complete list of supported formats see here (first column). I recommend using Cinepac as video codec and uncompressed audio. Reencoding of videos can be done, for example, using the freeware software VirtualDub.

Parameters

locationLocation of the video. This can be both a local file or a file on the internet.
If you encounter problems playing local videos or videos on the local network, you need to adjust your Java security settings. By default Java prohibits access to local files. In order to change this you need to add an exception to the Java Policy File:
  • Locate your policy-file (e.g. windows: "Program Files/Java/jre6/lib/security/java.policy")
  • Add this line to the GRANT-block: 
    permission java.io.FilePermission "PATH TO YOUR VIDEOS", "read";
    • For example: 
    ...
// default permissions granted to all domains

grant {
    permission java.io.FilePermission "z:\my_videos\*", "read";
...
Note that this needs to be done only once for each computer participating in the experiment.
