Examples

Global and automatically generated variables

If you want to use global variables, i.e. variables which can be accessed by all subjects, you can do so by prefixing a variable name with $, for example $game.

User defined variables

BoXS uses implicit and dynamic typing. You can assing both numbers and strings to a variable name without needing to specify the data type. BoXS will determine the correct type based on the context. If this is not possible, for example if you want to divide a number by a string, an error will be thrown.

Typing and undefined variables

Undefined variables are treated like 0 (double). The reason for this is that it is that it makes programs a little shorter and more readable. At the same time you have to watch out for typos as usually no error is thrown if you mistype a variable name.
