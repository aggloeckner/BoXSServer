Bonn Experiment System (BoXS)

The Bonn Experiment System is a software system designed to enable you to create and conduct experiments for laboratory environments as well as internet experiments. Its focus is on flexibility and ease of use. For more information please read the general information as well as the tutorial.

News

29.1.13: So far it has sometimes been problematic to include input elements in a loop. This issue has been resolved. It is now possible to write expressions like this:
for(i=0; i<5; i=i+1) { inputNumber(z[i]) }
29.1.13: The functionality of the assert-command has been changed in order to allow the experimenter to provide more feedback to the subject.
29.1.13: If you are writing long experiments for the BoXS, using the internal editor might not always be the best choice and you might want to use an external editor. For this case I hava created a basic language definition file which can be used for Sublime Text and other editors. You can download it here. After downloading it, just extract it to the respective packages folder.
28.1.13: You can now specify locales for your experiment which affects default texts as well as number formatting. See here for more information.
30.10.12: Even though not many new releases have been published to the homepage in the last few months, a lot of work has been going on regarding the BoXS. The whole server architecture hes been reworked and extensively tested within the last months. As an effect, experiments should run faster and more reliable.
One major new feature is the support for eye tracking devices which has been developed in cooperation with the Max Planck Institute for Research on Collective Goods. The eye tracking support has been successfully tested with some eye trackers. For more information, have a look at the documentation.
Another visual change is the use of the users' operating system's look and feel. This gives the BoXS a more native feel and in general looks a bit better.
There have been a lot more changes and bugfixes which will be documented in the changelog.
06.02.12: If you are interested in helping with the development of the BoXS, you can now access an official SVN-repository. Just contact me if you are interested in the details.
22.11.11: The Bonn Experiment System is hereby released as open source under the GNU Public License (GPL). You are free to download the original source code, expand and modify it to your contempt as long as you mention me as the original author and do not charge money for the use of the software. For more information click here.

Short Course or Tutorial

If you are interested in a short course / tutorial on using the Bonn Experiment System, feel welcome to contact me. I can give courses on the following topics:
  • Introduction to the BoXS: What are the main features? What are typical use cases?
  • Introduction to Programming in the BoXS: Based on an example, I show how to program and debug an experiment based on a given design and how to conduct the experiment.
  • Internet Experiments using the BoXS: I show how to program and conduct an internet experiment and give some general tipps on how (and when) to do internet experiments based on my own experience.
  • Advanced Programming Techniques: In this section I discuss frequent questions with regard to the BoXS and present how they can be solved. This can include topics like matching, loops and user interface design.
  • Error Handling and Experiment Recovery: Unfortunately, mistakes and crashes can happen. In this section I show how experiments can be resumed when something goes awry.
  • Setting up your own BoXS-Server: There are several reasons for setting up your own BoXS-Server, including an unreliable internet connection of your laboratory, low-latency experiments or special worries concerning data privacy. In this section I show how setting up a server can be done easily and reliably.
  • Developing with the BoXS: The BoXS is released as open source, which means that you can customise it to your specific needs. I show how to download the BoXS sourcecode and set up the BoXS repository. I give an overview over the implementation of the BoXS and show how to compile it and how to deploy a BoXS Server.
  • Discussion on the BoXS: Share your experiences with the BoXS, ask specific questions regarding your experiment, make feature requests and suggestions.

Contact

Please join the official Google Group/mailing list here. After joining you will be notified about all changes and updates to the BoXS. You can also participate in the discussion and send questions as well as requests to the list by sending emails. Please feel free to send:
  • Bug reports (if possible, including the program causing the bug as well as information on how to reproduce the problem)
  • Feature requests (for example experiment designs you have in mind)
  • All kinds of questions and documentation requests.
